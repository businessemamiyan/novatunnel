# panel-api

بک‌اند مستقل پنل مدیریت دسکتاپی NovaTunnel (فاز ۱A). یک سرویس FastAPI کاملاً جدا از
`admin_api/` (که ربات و Mini App تلگرام را سرویس می‌دهد) — هیچ کدی از `bot/`, `admin_api/`
یا `automation/` تغییر نکرده است. تنها چیز مشترک، پکیج `shared/` (منطق کسب‌وکار و اتصال
دیتابیس) است که هر دو سرویس مستقل از آن استفاده می‌کنند.

## چرا یک سرویس جدا (نه اضافه‌شدن به admin_api)

طبق تصمیم معماری این فاز، پنل مدیریت جدید **فقط روی سرور رلهٔ ایران (`109.122.246.148`)**
دیپلوی می‌شود، نه سرور هلند (`89.125.35.130`) که `bot`/`admin_api`/`automation` رویش زنده‌اند.
جدا نگه‌داشتن این سرویس یعنی:
- دیپلوی/ری‌استارت پنل جدید هیچ ریسکی برای بات/Mini App زنده ندارد.
- می‌توان panel-api را کاملاً مستقل، فقط روی سرور ایران بالا آورد.

## هویت و احراز هویت

`bot_admins` فقط `telegram_id` دارد، نه رمز عبور. به‌جای ساخت سیستم رمز جدید،
از **Telegram Login Widget** استفاده شده (مکانیزم رسمی تلگرام برای وب‌سایت‌ها — با initData
ووب‌اپ تلگرام که در `admin_api/auth.py` استفاده شده فرق دارد، نه معادل آن):
1. کاربر روی ویجت تلگرام در صفحه لاگین کلیک می‌کند.
2. تلگرام یک payload امضاشده (HMAC با `SHA256(bot_token)`) برمی‌گرداند.
3. `POST /api/auth/telegram-login` این امضا را تایید و عضویت در `bot_admins` را چک می‌کند.
4. یک جفت JWT (access کوتاه‌مدت + refresh بلندمدت) صادر می‌شود؛ refresh در جدول
   `admin_refresh_tokens` (فقط hash آن) ذخیره می‌شود تا قابل ابطال/چرخش باشد.

## راه‌اندازی محلی

```bash
cd panel-api
python -m venv venv && source venv/bin/activate
pip install -r requirements-dev.txt
cp .env.example .env   # مقادیر واقعی را پر کنید
PYTHONPATH=.. uvicorn main:app --reload --port 8001
```

> نکته: چون `shared/db.py` از `shared/config.py` استفاده می‌کند و آن ماژول
> `MARZBAN_ADMIN_USER`/`MARZBAN_ADMIN_PASSWORD` را الزامی می‌خواند (حتی اگر panel-api
> هرگز Marzban صدا نزند)، این دو متغیر باید در `.env` این سرویس هم وجود داشته باشند
> (می‌توانند مقدار واقعی یا هر رشتهٔ غیرخالی باشند).

## تست

```bash
PYTHONPATH=.. python -m pytest tests/ -v
```

فعلاً فقط `panel_auth` تست واحد دارد (تایید HMAC ویجت تلگرام + صدور/انقضای JWT) چون تنها
منطق امنیتی واقعاً جدید همین‌جاست؛ بقیهٔ endpointها مستقیماً توابع تست‌شدهٔ `shared/db.py`
را صدا می‌زنند.

## قبل از دیپلوی — باید تایید شود

این‌ها ریسک‌های شناخته‌شده و تست‌نشدهٔ دیپلوی روی سرور ایران هستند، نه فرض قطعی:
1. **دسترسی به Supabase از سرور ایران** — سرور ایران فعلاً فقط رلهٔ iptables اجرا می‌کند
   و اتصال خروجی‌اش به سرویس‌های خارجی (`aws-0-eu-west-1.pooler.supabase.com`) تست نشده.
2. **دسترسی به `api.telegram.org` از سرور ایران** — برای تایید ویجت لاگین نیازی به تماس
   شبکه‌ای نیست (HMAC آفلاین است)، ولی اگر بعداً endpointی نیاز به ارسال پیام تلگرام پیدا کرد
   (مثلاً اعلان تایید نماینده)، این تماس ممکن است از IP ایران فیلتر باشد.
3. **منابع سرور** — فقط ۷۶۴ مگابایت رم فیزیکی (+ ۲ گیگ swap موجود). یک پردازه uvicorn
   سبک‌وزن باید جواب بدهد ولی زیر بار واقعی تست نشده.
