# admin-panel

فرانت‌اند مستقل پنل مدیریت دسکتاپی NovaTunnel (فاز ۱A) — React + Vite + TypeScript + Tailwind،
RTL فارسی، هویت بصری دارک نئون همان برند NovaTunnel (رنگ‌ها/فونت مشترک با `miniapp/`).

IA (ساختار ناوبری سایدبار) از فایل نمونهٔ `admin-v2` که کاربر ارائه داد الهام گرفته شده،
منطبق‌شده با نیاز واقعی NovaTunnel (فروش/نمایندگان/مالی به‌جای محصولات/سفارش‌های عمومی).

## چه چیزی در فاز ۱A ساخته شده

- ورود با **Telegram Login Widget** (نه initData ووب‌اپ که Mini App استفاده می‌کند)
- داشبورد KPI (فروش، حساب‌ها، حجم، نمایندگان، موارد در انتظار تایید)
- مدیریت مشتریان: جست‌وجو، صفحه‌بندی، خروجی اکسل، جزئیات مشتری، اعطای دستی حجم هدیه

بخش‌های نمایندگان/بسته‌ها/سفارش‌ها/مالی/تنظیمات در سایدبار به‌صورت «به‌زودی» دیده می‌شوند —
این‌ها فاز ۱B/۱C هستند، هنوز صفحه‌ای پشتشان نیست.

## راه‌اندازی محلی

```bash
cd admin-panel
npm install
cp .env.example .env   # آدرس panel-api و یوزرنیم بات را تنظیم کنید
npm run dev
```

باید هم‌زمان `panel-api` روی پورت ۸۰۰۱ در حال اجرا باشد.

## Build

```bash
npm run build   # خروجی در dist/
```

## نکات دیپلوی (سرور رلهٔ ایران)

طبق تصمیم معماری این فاز، هم `panel-api` و هم build استاتیک این فرانت **فقط روی سرور
رلهٔ ایران (`109.122.246.148`)** سرو می‌شوند، نه سرور هلند. `dist/` را می‌توان با هر وب‌سروری
(nginx ساده) یا حتی static mount خودِ panel-api سرو کرد. ریسک‌های تست‌نشدهٔ این تصمیم
(دسترسی سرور ایران به Supabase/Telegram API) در `panel-api/README.md` مستند شده‌اند.
