# پرامپت برای Claude Code: افزودن پنل ادمین به Mini App موجود NovaTunnel

## هدف این تسک
افزودن یک **بخش پنل ادمین** به همان Mini App موجود (نه اپلیکیشن جدا) که فقط برای کاربری با `telegram_id` منطبق بر `ADMIN_TELEGRAM_ID` قابل مشاهده باشد. این پنل باید از طریق تلگرام (چه موبایل چه دسکتاپ) توسط خود من (مالک سیستم) در دسترس باشد — چون Mini App روی هر دستگاهی که تلگرام نصب باشد باز می‌شود، نیازی به دامنه یا لاگین جداگانه نیست؛ فقط باید تشخیص هویت ادمین بر اساس `telegram_id` انجام شود.

## ویژگی‌های مورد نیاز

### ۱. شمارنده فروش (Sales Counter)
یک ویجت در بالای پنل ادمین که نمایش دهد:
- تعداد کل فروش امروز / این هفته / این ماه
- مجموع مبلغ فروش (تومان) در همان بازه‌ها
- منبع داده: جدول `purchases` (طبق طراحی موجود در `NovaTunnel-Business-Rules.md`، بخش ۹)

### ۲. بخش حسابداری / هزینه‌ها (Accounting)
یک بخش جدید که به من اجازه بدهد:
- هزینه‌ای که از پول فروش برداشت و خرج کرده‌ام را ثبت کنم (مبلغ، توضیح کوتاه، تاریخ، دسته‌بندی اختیاری مثل «سرور»، «برداشت شخصی»، «تبلیغات»)
- لیست تمام هزینه‌های ثبت‌شده را با تاریخ ببینم
- یک خلاصه ساده: مجموع فروش − مجموع هزینه‌ها = سود خالص فعلی

باید یک جدول جدید در Supabase طراحی و ایجاد شود، مثلاً:
```sql
CREATE TABLE expenses (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  amount_toman BIGINT NOT NULL,
  description TEXT NOT NULL,
  category TEXT,
  created_by UUID REFERENCES users(id),
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```
(اگر ستون یا نام بهتری منطبق با اسکیمای فعلی پروژه لازم است، خودت با توجه به کانونشن‌های موجود اصلاح کن.)

فرم ثبت هزینه باید ساده باشد: یک عدد (مبلغ) + یک متن (توضیح) + دکمه ثبت. بعد از ثبت، بلافاصله در لیست و در خلاصه سود/زیان منعکس شود.

### ۳. دکمه کپی شماره کارت (در بخش پرداخت مشتری، نه پنل ادمین)
در همان صفحه‌ای از Mini App که شماره کارت بانکی برای پرداخت کارت‌به‌کارت به مشتری نمایش داده می‌شود، یک دکمه کوچک «کپی» کنار شماره کارت اضافه شود که با یک کلیک شماره را در کلیپ‌بورد کاربر کپی کند و یک پیام تأیید کوتاه (toast) نشان دهد. این بخش را در فایل موجود Mini App پیدا کن (اگر بخش نمایش شماره کارت هنوز اصلاً پیاده‌سازی نشده، آن را هم بساز).

## نکات فنی مهم برای پیاده‌سازی
- تشخیص هویت ادمین باید سمت بک‌اند (n8n یا مستقیم در فراخوانی Supabase) چک شود، نه فقط مخفی‌کردن ظاهری در فرانت — چون Mini App کد frontend را همه می‌بینند، اگر فقط با CSS/JS مخفی شود هرکسی می‌تواند با DevTools ببیندش. حداقل باید API/queryهای مرتبط با این پنل، `telegram_id` را چک کنند و برای غیر ادمین خطا برگردانند.
- هماهنگ با تم بصری موجود Mini App (رنگ‌ها: `--cyan #00e5ff`, `--purple #a855f7`, `--bg #0b0e1a`, فونت Vazirmatn, RTL کامل) پیش برو — همان متغیرهای CSS موجود در فایل فعلی را استفاده کن، دوباره تعریف نکن.
- اگر برای ثبت هزینه یا خواندن آمار فروش نیاز به endpoint جدید در n8n است، آن را هم بساز و مستند کن.
- خروجی نهایی باید شامل: تغییرات در فایل HTML/CSS/JS مینی‌اپ، اسکریپت SQL برای جدول(های) جدید Supabase، و در صورت نیاز workflow جدید یا اصلاح‌شده n8n باشد.

## چیزی که لازم نیست بسازی (خارج از اسکوپ این تسک)
- سیستم مقاومت در برابر قطعی اینترنت ملی (این یک تسک زیرساختی جداست، بعداً جدا برنامه‌ریزی می‌شود)
- سیستم نمایندگی یا رفرال (قبلاً طراحی شده یا در حال طراحی جداست)
## سیستم  حسابداری هم برا پنل نمایندگی لازمه
-دقیقا سیستم حسابداری که برای ادمین میسازی برای نمایندگی هم بسازی یعنی پنل نمایندگی هم حساب خرید و فروشش رو داشته باشه
## قبل از شروع
اگر ساختار دقیق فایل فعلی Mini App یا نام دقیق جداول Supabase (`purchases`, `users`) با آنچه در اسناد پروژه آمده فرق دارد، لطفاً اول فایل‌های موجود پروژه را بخوان و با همان ساختار واقعی هماهنگ کن، نه صرفاً بر اساس مستندات.
